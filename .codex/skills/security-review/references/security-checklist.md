# Security Checklist

현재 이슈의 변경 범위와 관련 있는 항목만 검토한다. 요구사항의 원천은 `docs/spec/spec-fixed.md`, 기능 PRD/TRD, architecture 문서, 이슈 AC다.

## 인증과 세션

- 서버 세션과 쿠키 기준 인증을 유지한다.
- JWT Access/Refresh Token은 MVP 기본 인증으로 도입하지 않는다.
- 로그인, 로그아웃, 이메일 인증, 비밀번호 재설정 흐름에서 토큰은 평문 저장하지 않는다.
- 인증 전 사용자와 미완료 계정은 허용된 최소 기능만 접근할 수 있다.
- 세션 식별자, 인증 토큰, 비밀번호 재설정 토큰, 이메일 인증 토큰을 로그나 응답에 노출하지 않는다.

## 인가와 소유권

- 사용자 입력의 `userId`, `senderId`, 닉네임, 권한 값을 신뢰하지 않는다.
- 요청 주체는 서버 세션과 서버 저장 상태로 판정한다.
- 타 사용자 데이터, 덱, 컬렉션, 경기, 기보, 게시글 원문, 신고, 관리자 기능 접근은 소유권 또는 역할을 검증한다.
- 관리자 API와 관리자 원문 조회는 `ADMIN` 역할과 감사 로그를 요구한다.
- 차단/제재/계정 상태가 기능 접근에 영향을 주는 요구사항을 빠뜨리지 않는다.

## REST와 오류 응답

- REST API는 서버 세션 기반 인증을 전제로 한다.
- 오류 응답은 내부 구현, SQL, stack trace, 보안 정책 세부를 노출하지 않는다.
- 입력 검증 실패는 도메인 상태를 변경하지 않는다.
- 상태 변경 요청에서 CSRF, CORS, SameSite 쿠키 정책과 충돌하는 구현을 만들지 않는다.
- rate limit이 요구된 요청은 IP, 계정, 이메일 등 명세 기준 키를 확인한다.

## STOMP와 실시간 경기

- STOMP `CONNECT`, `SUBSCRIBE`, `SEND`마다 인증, 권한, 계정 상태, 제재를 검증한다.
- 개인 채널 라우팅은 클라이언트가 보낸 userId가 아니라 인증 세션 기준으로 수행한다.
- 클라이언트가 보낸 게임 상태, 카드 효과, 턴, 타이머, 승패, 보상, 권한 값을 서버가 신뢰하지 않는다.
- 거부된 명령은 도메인 상태를 변경하지 않는다.
- 메시지 크기, 과도한 SEND, 인증 실패에 대한 제한과 로그 요구사항을 확인한다.

## 데이터와 저장

- SQL/JPQL/native query/동적 정렬/검색 조건에서 주입 가능성을 확인한다.
- 비밀번호, 이메일 인증 토큰, 비밀번호 재설정 토큰은 안전한 해시와 만료/무효화 정책을 따른다.
- 민감 데이터 projection은 최소화한다.
- 감사 또는 보안 로그 대상 행위는 누락하지 않는다.
- DB 수동 관리자 권한 부여 전제와 충돌하는 자동 승격 로직을 만들지 않는다.

## 운영과 관측성

- Actuator 상세 health 정보는 내부망 또는 인증된 관리자/모니터링 주체만 접근해야 한다.
- Prometheus, health, metrics endpoint가 외부에 과도한 정보를 노출하지 않는지 확인한다.
- 보안 로그에는 필요한 사건은 남기되 민감값 원문은 남기지 않는다.
- 파일 업로드, 외부 API, JWT, 다중 인스턴스 확장은 MVP 범위 밖이면 현재 이슈에서 임의 도입하지 않는다.

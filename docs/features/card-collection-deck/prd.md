# Card Collection & Deck PRD

## 목적

사용자가 스타터 카드와 카드팩으로 카드를 획득하고 보유 수량을 관리하며, 보유 카드와 중복 제한을 만족하는 덱을 편성해 빠른 매치에 사용할 수 있도록 한다.

## 대상 사용자

- 신규 카드와 기본 덱을 지급받는 사용자
- 기본 카드팩을 개봉해 컬렉션을 확장하려는 사용자
- 보유 카드로 빠른 매치용 덱을 편성·변경하려는 사용자

## 범위

### 포함

- `docs/cards`에 저장되는 카드 이미지·이름·조건·효과·마나 비용 정보와 카드 목록 제공
- 신규 사용자 스타터 컬렉션과 기본 덱 지급
- 기본 카드팩 1종의 0골드 개봉
- 카드팩 3장 지급과 슬롯별 레전드 확률
- KST 기준 일일 성공 개봉 제한
- 중복 카드 보유 수량 누적
- 카드팩 개봉 멱등성·동시성·감사 기록
- 레전드 1장과 일반 카드 5장으로 구성되는 덱
- 보유 수량·덱 중복 제한·카드 종류에 따른 덱 유효성 검증
- 빠른 매치 진입과 경기 생성 직전의 덱 재검증
- 영구적인 카드 식별자와 배포 후 변경되지 않는 카드 버전의 관리
- 카드별 경기 투입·승리 통계에 사용할 카드 식별과 버전 정보의 관리

### 제외

- 경기 중 카드 효과 적용과 카드 사용 타이밍
- 유료 골드 구매, 출석·경기 보상, 카드팩 추가
- 천장·확률 보정
- 카드 효과·확률·팩 가격 관리자 편집
- 카드 정의의 기술적 저장·배포 방식

## 주요 사용자 흐름

1. 신규 사용자가 계정 활성화 후 스타터 컬렉션과 기본 덱을 받는다.
2. 사용자가 기본 카드팩을 개봉해 카드 3장을 받는다.
3. 서버가 슬롯별 레전드 확률과 일일 개봉 제한을 검증·적용한다.
4. 사용자가 컬렉션의 보유 수량을 확인한다.
5. 사용자가 레전드 1장과 일반 카드 5장으로 덱을 구성한다.
6. 서버가 보유 수량과 덱 규칙을 검증하고 유효 덱으로 저장한다.
7. 빠른 매치 진입과 경기 생성 직전에 덱 유효성을 다시 확인한다.

## 상세 사용자 시나리오

### 시나리오 1. 신규 사용자 스타터 지급

- 시작 조건: 주요 기능 사용 조건을 충족한 신규 사용자이며 스타터 지급 기록이 없다.
- 사용자 행동: 사용자가 자신의 컬렉션과 기본 덱을 조회한다.
- 시스템 반응: 서버는 스타터 레전드 1장, 일반 카드 5장으로 구성된 기본 덱과 즉시 플레이 가능한 컬렉션을 한 번 지급한다.
- 사용자에게 보이는 결과: 사용자는 기본 덱과 초기 카드 보유 수량을 확인하고 빠른 매치 준비를 할 수 있다.
- 완료 조건: 같은 사용자에게 새로고침·재요청으로 스타터 카드와 덱이 중복 지급되지 않는다.

### 시나리오 2. 기본 카드팩 개봉

- 시작 조건: 사용자가 당일 성공 개봉 10회 미만이고 기본 카드팩을 개봉할 수 있는 상태다.
- 사용자 행동: 사용자가 기본 카드팩 개봉을 요청한다.
- 시스템 반응: 서버는 0골드 가격, 일일 제한, 요청 멱등성을 확인하고 세 슬롯을 독립 판정한다. 각 슬롯은 레전드 1% 확률을 적용하고, 결과 카드 3장·보유 수량·확률 버전·감사 기록을 확정한다.
- 사용자에게 보이는 결과: 사용자는 카드 3장과 갱신된 컬렉션 수량을 확인한다.
- 완료 조건: 클라이언트가 제출한 카드 ID·확률·결과는 사용되지 않으며, 중복 카드는 수량으로 누적된다.

### 시나리오 3. 일일 제한과 동시 개봉

- 시작 조건: 사용자의 당일 성공 개봉 횟수가 9회이고 여러 개봉 요청이 동시에 도착한다.
- 사용자 행동: 사용자가 동시에 카드팩 개봉을 요청한다.
- 시스템 반응: 서버는 성공 개봉 횟수를 원자적으로 확인해 최대 10회까지만 성공시킨다. 초과 요청은 일일 제한 초과로 처리한다.
- 사용자에게 보이는 결과: 성공한 개봉 결과만 컬렉션에 반영되고, 제한을 초과한 요청에는 제한 안내가 표시된다.
- 완료 조건: 하루 성공 개봉이 10회를 넘지 않고, 실패 요청으로 카드·골드·개봉 횟수가 추가 변경되지 않는다.

### 시나리오 4. 보유 카드로 유효 덱 편성

- 시작 조건: 사용자가 레전드 카드와 일반 카드의 보유 수량을 가지고 있다.
- 사용자 행동: 사용자가 레전드 1장과 일반 카드 5장을 선택해 덱을 저장한다.
- 시스템 반응: 서버는 카드 종류, 보유 수량, 덱 중복 제한과 필수 카드 수를 검증한 뒤 유효한 덱만 저장한다.
- 사용자에게 보이는 결과: 사용자는 저장된 덱과 빠른 매치에 사용할 수 있는 유효 상태를 확인한다.
- 완료 조건: 덱에 없는 카드나 보유 수량을 초과한 카드가 저장되지 않는다.

### 시나리오 5. 경기 직전 덱 무효화

- 시작 조건: 사용자가 대기열 진입 후 덱을 변경했거나 카드 상태가 변경되어 경기 생성 직전 덱이 유효하지 않다.
- 사용자 행동: 서버가 매칭된 사용자의 경기 생성을 최종 검증한다.
- 시스템 반응: 서버는 덱 유효성 실패를 확인하고 경기 생성을 진행하지 않는다. 이미 다른 경기에서 고정된 덱 스냅샷에는 영향을 주지 않는다.
- 사용자에게 보이는 결과: 사용자는 경기 시작 실패와 덱 재확인 필요 안내를 확인한다.
- 완료 조건: 유효하지 않은 덱으로 경기가 생성되지 않는다.

## 상태/권한별 엣지 케이스

- 주요 기능을 사용할 수 없는 계정은 카드팩 개봉과 덱 편성을 할 수 없다.
- 스타터 지급은 사용자별 멱등 처리하며 이미 지급된 사용자에게 중복 지급하지 않는다.
- 기본 카드팩은 0골드이고 카드팩 1개는 카드 3장을 지급한다.
- 각 카드 슬롯은 독립적으로 레전드 1%를 판정하며 0~3장의 레전드가 모두 가능하다.
- 일일 성공 개봉 제한은 KST 기준 00:00:00에 초기화된다.
- 중복 요청은 같은 결과를 재사용할 수 있고, 다른 내용의 같은 요청 ID는 충돌로 거부할 수 있다.
- 개봉 실패는 카드 지급·보유 수량·성공 횟수·감사 결과를 성공 상태로 변경하지 않는다.
- 중복 카드는 자동 환급·확률 보정 없이 보유 수량에 누적한다.
- 덱은 레전드 1장과 일반 카드 5장으로 구성하며 보유 수량을 초과할 수 없다.
- 동일 일반 카드는 덱 안에 최대 2장까지 포함할 수 있다.
- 매칭 진입 시점과 경기 생성 직전에 덱 유효성을 다시 확인한다.
- 경기 생성 후 덱 변경은 이미 고정된 경기 덱 스냅샷에 영향을 주지 않는다.
- 경기 생성 시 덱에 포함된 카드의 식별자와 버전이 함께 고정된다.
- 같은 카드 버전의 의미와 공개 계약은 배포 후 바꾸지 않으며, 의미가 달라지면 새 버전을 사용한다.
- 사용 전 레전드 카드는 상대의 경기 상태·이벤트·재접속 스냅샷에 포함하지 않는다.
- 카드별 경기 투입 수와 승리 수 통계는 카드 식별과 당시의 카드 버전을 기준으로 구분할 수 있어야 한다.

## 사용자에게 보이는 결과

- 카드 이미지·이름·조건·효과·마나 비용 정보
- 스타터 컬렉션과 기본 덱 지급 결과
- 카드팩 개봉 결과 3장과 컬렉션 수량 변화
- 일일 개봉 횟수와 제한 초과 안내
- 중복 카드 수량 누적 결과
- 덱 구성, 유효성 검증 결과와 저장 결과
- 경기 직전 덱 검증 실패 안내
- 상대에게 공개되는 일반 카드 정보와 비공개 레전드 정보의 구분

## 수용 기준

- 신규 사용자에게 스타터 컬렉션과 기본 덱이 멱등적으로 지급된다.
- `docs/cards`에 MVP 카드 12장의 이미지·이름·발동 조건·효과·효과 실패 조건·마나 비용·식별자·버전 정보가 정의된다.
- 기본 카드팩 1종을 0골드로 개봉할 수 있다.
- 카드팩은 카드 3장을 지급하고 각 슬롯에 레전드 1% 확률을 독립 적용한다.
- KST 기준 하루 성공 개봉은 10회로 제한되며 동시 요청으로 초과할 수 없다.
- 중복 카드는 보유 수량으로 누적되고 자동 환급·천장·확률 보정은 없다.
- 카드팩 결과와 확률 버전이 감사 기록으로 남는다.
- 덱은 레전드 1장과 일반 카드 5장으로 구성되며, 동일 일반 카드는 최대 2장까지 포함한다.
- 보유하지 않은 카드 또는 보유 수량을 초과한 카드는 덱에 저장할 수 없다.
- 빠른 매치 진입과 경기 생성 직전에 덱 유효성이 검증된다.
- 경기 생성 시 덱 스냅샷이 고정된다.
- 경기 덱에는 카드 식별자와 버전이 함께 고정되며, 같은 버전의 의미는 배포 후 변경되지 않는다.
- 사용 전 레전드 카드가 상대에게 노출되지 않는다.
- 카드별 경기 투입 수·승리 수·승률 분석에 필요한 카드 식별과 버전 정보가 종료 경기 기록과 연결된다.

## 미확정 사항

- 카드별 통계의 사용자 노출 범위와 카드 밸런스 판단 기준

MVP 카드 12장의 제품 내용과 카드 식별자·버전 정책은 확정되었다. MVP 이후 카드팩 종류와 상품 정책은 별도 제품 검토 대상으로 두며, 카드 정의와 확률 버전의 저장·배포 및 과거 경기 연결 방식은 후속 아키텍처·기술 문서에서 결정한다.

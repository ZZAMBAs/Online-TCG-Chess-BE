# Community PRD

## 목적

로그인 사용자가 게시글을 작성·조회·검색하고 자신의 게시글을 관리할 수 있도록 하며, 게시글 상태와 금칙어 정책에 따라 일반 사용자에게 안전한 콘텐츠만 노출한다.

## 대상 사용자

- 커뮤니티 게시글을 작성·조회·검색하려는 로그인 사용자
- 자신의 게시글을 수정·삭제하려는 작성자
- 숨김·삭제 상태의 게시글에 접근을 시도하는 사용자

## 범위

### 포함

- 게시글 작성, 목록, 상세, 수정, 삭제
- 제목·내용·작성자 닉네임 검색
- `ACTIVE`, `DELETED`, `AUTO_HIDDEN`, `MANUAL_HIDDEN` 상태
- 게시글 원문과 마스킹 결과 관리
- 사용자 화면의 plain text·HTML escape·금칙어 마스킹
- 신고 기능과 신고에 따른 자동 숨김의 제품 결과
- 관리자 숨김·복구 결과와 운영 기능 연계

### 제외

- 댓글·게시글 좋아요·댓글 좋아요
- 이미지 업로드
- 전문 검색 엔진·고급 검색 랭킹
- 허위 신고 자동 제재와 이의 제기 절차
- 신고 검토·제재·금칙어 사전 관리의 상세 운영 흐름

## 주요 사용자 흐름

1. 로그인 사용자가 제목과 본문을 입력해 게시글을 작성한다.
2. 서버가 계정 상태·제재·입력·금칙어를 검증하고 마스킹 결과를 노출한다.
3. 사용자가 활성 게시글 목록·상세·검색 결과를 조회한다.
4. 작성자는 자신의 게시글을 수정·삭제한다.
5. 신고 누적 또는 관리자 판단으로 게시글이 숨김 상태가 된다.
6. 숨김·삭제 게시글은 일반 목록과 검색에서 제외되고, 관리자 복구 시 다시 노출될 수 있다.

## 상세 사용자 시나리오

### 시나리오 1. 게시글 작성과 마스킹 표시

- 시작 조건: 로그인된 사용자가 커뮤니케이션 정지나 계정정지 상태가 아니다.
- 사용자 행동: 사용자가 제목과 본문을 작성해 게시글을 등록한다.
- 시스템 반응: 서버는 작성 권한을 확인하고 금칙어를 게시글 작성 시점의 사전 기준으로 마스킹한 결과를 사용자 API에 반환한다. 원문과 마스킹 결과를 보관한다.
- 사용자에게 보이는 결과: 사용자는 게시글 목록과 상세에서 HTML이 실행되지 않고 금칙어가 마스킹된 내용을 확인한다.
- 완료 조건: 게시글이 `ACTIVE`로 노출되고 사용자 화면에 원문이 노출되지 않는다.

### 시나리오 2. 게시글 수정·삭제

- 시작 조건: 사용자가 자신의 `ACTIVE` 게시글을 가지고 있다.
- 사용자 행동: 작성자가 게시글을 수정하거나 삭제한다.
- 시스템 반응: 서버는 작성자 소유권과 계정 제재를 확인하고, 수정 시 새 입력에 마스킹을 적용하며 삭제 시 `DELETED` 상태로 전환한다.
- 사용자에게 보이는 결과: 수정된 마스킹 내용이 보이고, 삭제된 게시글은 일반 목록·검색에서 사라진다.
- 완료 조건: 다른 사용자의 게시글은 변경되지 않고 삭제된 게시글이 일반 사용자에게 다시 노출되지 않는다.

### 시나리오 3. 게시글 검색과 숨김 상태 제외

- 시작 조건: 여러 게시글이 있고 일부는 `ACTIVE`, 일부는 숨김 또는 삭제 상태다.
- 사용자 행동: 로그인 사용자가 제목·내용·작성자 닉네임으로 검색한다.
- 시스템 반응: 서버는 활성 게시글만 대상으로 단순 검색을 수행하고 결과를 마스킹된 내용으로 제공한다.
- 사용자에게 보이는 결과: 검색 결과에는 활성 게시글만 보이며 숨김·삭제 게시글은 검색되지 않는다.
- 완료 조건: 숨김·삭제 콘텐츠의 원문과 존재 여부가 일반 검색 결과에 노출되지 않는다.

### 시나리오 4. 신고 누적에 따른 자동 숨김

- 시작 조건: `ACTIVE` 게시글에 서로 다른 유효 사용자들의 신고가 누적되고 있다.
- 사용자 행동: 유효 사용자들이 게시글을 신고한다.
- 시스템 반응: 서버는 사용자별 중복 신고를 막고, 서로 다른 유효 사용자 5명의 신고가 누적되면 게시글을 한 번만 `AUTO_HIDDEN`으로 전환한다.
- 사용자에게 보이는 결과: 임계치에 도달한 게시글은 일반 목록·검색에서 사라지고 숨김 안내만 제공된다. 신고 수와 진행 상황은 일반 사용자에게 공개하지 않는다.
- 완료 조건: 동시 신고에서도 자동 숨김이 한 번만 적용되고 신고가 임계치를 우회하지 않는다.

### 시나리오 5. 제재 상태에서의 게시글 변경 거부

- 시작 조건: 사용자가 커뮤니케이션 정지 또는 계정정지 상태다.
- 사용자 행동: 사용자가 게시글 작성·수정·삭제 또는 신고를 요청한다.
- 시스템 반응: 서버는 계정 상태와 제재를 확인해 요청을 거부한다. 계정정지 사용자는 커뮤니티 접근 자체가 제한된다.
- 사용자에게 보이는 결과: 제한 사유를 확인하고 게시글 상태는 변경되지 않는다.
- 완료 조건: 제재 사용자가 게시글 변경·신고를 우회할 수 없다.

## 상태/권한별 엣지 케이스

- 커뮤니티 조회와 작성은 로그인 사용자만 가능하다.
- 계정정지 사용자는 커뮤니티를 사용할 수 없다.
- 커뮤니케이션 정지 사용자는 작성·수정·삭제·신고를 할 수 없다.
- 게시글 작성자는 자신의 게시글만 수정·삭제할 수 있다.
- 관리자의 숨김·복구는 별도 관리자 권한과 운영 흐름을 따른다.
- `DELETED`, `AUTO_HIDDEN`, `MANUAL_HIDDEN` 게시글은 일반 목록·검색에서 제외한다.
- 숨김·삭제 게시글의 직접 접근은 거부하거나 숨김 안내만 제공한다.
- 제목·내용·작성자 닉네임 검색은 `ACTIVE` 게시글만 대상으로 한다.
- 사용자 API는 원문이 아닌 마스킹 결과를 반환한다.
- 금칙어는 게시글 생성·수정 시점의 사전 기준으로 처리하고, 사전 변경을 기존 게시글에 소급하지 않는다.
- 같은 사용자는 같은 게시글을 한 번만 신고할 수 있다.
- 숨김·삭제 게시글은 신고할 수 없다.
- 신고 5건은 서로 다른 유효 사용자 기준이며 동시 신고에서도 자동 숨김은 한 번만 적용한다.
- 댓글·좋아요 요청은 MVP에서 제공하지 않는다.

## 사용자에게 보이는 결과

- 게시글 작성·수정·삭제 결과
- 마스킹된 제목·본문과 HTML이 실행되지 않는 plain text 표시
- 활성 게시글 목록·상세·검색 결과
- 숨김·삭제 게시글의 접근 제한 안내
- 신고 접수와 자동 숨김 결과
- 제재 상태에 따른 게시글 작업 거부 안내

## 수용 기준

- 로그인 사용자만 게시글을 조회·작성할 수 있다.
- 게시글 작성·수정·삭제·검색·상세 조회가 가능하다.
- 작성자는 자신의 게시글만 수정·삭제할 수 있다.
- 게시글 상태가 `ACTIVE`, `DELETED`, `AUTO_HIDDEN`, `MANUAL_HIDDEN`으로 관리된다.
- 숨김·삭제 게시글은 일반 목록과 검색에서 제외된다.
- 제목·내용·작성자 닉네임 검색은 활성 게시글만 반환한다.
- 사용자 화면에는 금칙어 마스킹 결과만 표시된다.
- 원문과 마스킹 결과가 게시글 생성·수정 시점 기준으로 보관된다.
- 서로 다른 유효 사용자 5명의 신고가 누적되면 게시글이 자동 숨김 처리된다.
- 동일 사용자의 중복 신고와 동시 신고 경쟁이 정책에 맞게 처리된다.
- 커뮤니케이션 정지·계정정지 사용자의 게시글 작업과 신고가 제한된다.
- 댓글과 좋아요는 제공하지 않는다.

## 미확정 사항

- 게시글 제목·본문 길이와 검색 입력 길이
- 게시글 목록의 페이지 크기·정렬·검색어 일치 방식
- 숨김·삭제 게시글 직접 접근 시 세부 안내 문구
- 관리자 복구 후 게시글이 원래 위치에 노출되는지 여부

금칙어 필터의 고급 우회 탐지와 외부 검색·필터 서비스는 MVP 이후로 넘긴다. 게시글 endpoint, payload, 저장 schema와 검색 구현 방식은 기술 문서로 넘긴다.

# Gameplay PRD

## 목적

매칭이 완료된 두 플레이어가 서버가 권위적으로 판단하는 체스와 카드 규칙에 따라 실시간 경기를 진행하고, 유효한 행동·시간·재접속·종료 결과를 일관되게 경험하도록 한다.

## 대상 사용자

- 매칭이 완료되어 경기에 참가한 백·흑 플레이어
- 연결이 끊겼다가 경기로 복귀하는 플레이어

## 범위

### 포함

- 경기 시작 스냅샷과 서버 권위 상태
- 표준 8x8 체스와 캐슬링·앙파상·프로모션
- 체크·체크메이트·스테일메이트·기물 부족·50수·3회 반복
- 턴 시작, 이동 전 카드 행동, 말 이동, 프로모션, 이동 후 행동, 턴 종료
- 일반 카드·레전드 카드 사용과 공개·비공개 정보 구분
- 마나 획득·소비·상한과 체크 관련 마나 규칙
- 개인 시계, 시간패, 연결 끊김, 재접속, 몰수패
- 항복과 경기 종료 사유
- 서버 이벤트, 스냅샷·재동기화, 멱등 행동 처리
- 경기별 명령 직렬화와 종료 결과 단일 확정
- 카드 사용과 체크메이트 기물 통계를 위한 원천 경기 기록

### 제외

- 카드팩 개봉, 카드 보유 수량, 스타터 컬렉션
- 덱 생성·수정·유효성의 제품 흐름
- 경기 결과 전적·MMR 저장과 기보 조회 화면
- 경기 채팅
- 관전·친선 코드·합의 무승부
- 분산 타이머와 다중 인스턴스 경기 복구

## 주요 사용자 흐름

1. 서버가 매칭된 두 플레이어의 색, 덱 스냅샷, 초기 경기 상태를 제공한다.
2. 백부터 서버 기준 첫 턴을 시작한다.
3. 현재 플레이어가 턴 시작 후 이동 전 카드 행동을 선택한다.
4. 게임이 끝나지 않으면 플레이어가 합법적인 말을 이동하고, 필요하면 프로모션을 선택한다.
5. 플레이어가 이동 후 카드 사용·드로우·패스 중 하나를 선택한다.
6. 서버가 상태와 이벤트 버전을 확정한 뒤 다음 턴으로 넘긴다.
7. 시간패·몰수패·항복·체크메이트·무승부 조건이 발생하면 즉시 경기를 종료한다.
8. 재접속한 플레이어에게 최신 권위 스냅샷을 제공한다.

## 상세 사용자 시나리오

### 시나리오 1. 정상 턴 진행

- 시작 조건: 매칭된 경기가 시작되었고 백 플레이어의 첫 턴이다.
- 사용자 행동: 현재 플레이어가 이동 전 행동을 선택하고 합법적인 말을 이동한 뒤 이동 후 행동을 확정한다.
- 시스템 반응: 서버는 턴 단계, 현재 플레이어, 체스 합법성, 카드 조건, 마나, 킹 안전성을 순서대로 검증하고 각 확정 상태의 버전을 증가시킨다.
- 사용자에게 보이는 결과: 양쪽 플레이어는 공개 경기 상태와 다음 턴 정보를 확인한다. 개인 채널에는 자신의 비공개 카드 정보가 전달된다.
- 완료 조건: 한 턴의 각 단계가 한 번씩 처리되고 다음 플레이어에게 턴이 넘어간다.

### 시나리오 2. 카드 사용으로 즉시 경기 종료

- 시작 조건: 현재 플레이어가 카드 사용 조건과 마나를 충족하고, 카드 효과가 상대를 체크메이트 상태로 만든다.
- 사용자 행동: 플레이어가 이동 전 또는 이동 후 카드 사용을 요청한다.
- 시스템 반응: 서버는 최종 보드 상태를 기준으로 체크메이트를 판정하고 말 이동·이동 후 행동 또는 다음 턴으로 진행하지 않고 경기를 종료한다.
- 사용자에게 보이는 결과: 양쪽 플레이어는 카드 효과 결과와 경기 종료·승패를 확인한다.
- 완료 조건: 카드 효과가 체스 승패 판정을 우회하지 않고 경기 종료 이벤트가 한 번만 확정된다.

### 시나리오 3. 잘못된 행동 거부와 재시도

- 시작 조건: 플레이어가 자기 턴이 아니거나 현재 단계와 맞지 않는 행동을 제출한다.
- 사용자 행동: 플레이어가 잘못된 이동, 부족한 마나의 카드 사용, 체크를 해소하지 못하는 카드 사용 또는 중복 행동을 요청한다.
- 시스템 반응: 서버는 행동을 거부하고 실패 사유와 가능한 경우 최신 상태 버전을 반환한다. 보드, 손패, 마나, 턴 단계, 타이머 결과와 경기 결과는 변경하지 않는다.
- 사용자에게 보이는 결과: 실패한 행동이 반영되지 않고 최신 서버 상태로 복귀하며, 같은 단계에서 유효한 행동을 다시 시도할 수 있다.
- 완료 조건: 실패 명령은 도메인 상태와 `GameEvent`를 변경하지 않는다. 단, 멱등성·보안·실패 기록은 별도로 남을 수 있다.

### 시나리오 4. 프로모션 선택 대기

- 시작 조건: 표준 말 이동으로 폰이 끝 행에 도달했다.
- 사용자 행동: 플레이어가 퀸·룩·비숍·나이트 중 하나를 선택하거나 선택 전 다른 행동을 요청한다.
- 시스템 반응: 서버는 프로모션 선택 전에는 이동 후 행동, 턴 종료, 상대 행동을 허용하지 않는다. 선택이 완료되면 최종 보드 기준으로 체크와 경기 종료 조건을 다시 판정한다.
- 사용자에게 보이는 결과: 승격 기물 선택 화면과 선택 결과를 확인한다.
- 완료 조건: 표준 말 이동의 프로모션만 발생하고 카드 효과로 폰이 끝 행에 도달한 경우에는 프로모션이 발생하지 않는다.

### 시나리오 5. 연결 끊김과 재접속

- 시작 조건: 진행 중인 경기에서 한 플레이어의 연결이 끊겼다.
- 사용자 행동: 플레이어가 20초 안에 재접속하거나, 복귀하지 않은 채 유예 시간이 지난다.
- 시스템 반응: 서버는 연결 중단 상태에서도 본인 턴의 시계를 계속 감소시키고, 재접속 시 재접속 전과 동일한 게임 화면에서 최신 스냅샷을 제공한다. 20초 안에 복귀하지 않으면 몰수패를 확정한다.
- 사용자에게 보이는 결과: 재접속한 사용자는 동일한 게임 화면에서 최신 보드·턴·카드·타이머 상태를 받고, 늦게 복귀하면 몰수패 결과를 확인한다.
- 완료 조건: 이미 확정된 시간패·몰수패 결과는 재접속으로 되돌아가지 않는다.

### 시나리오 6. 시간패와 종료 경쟁

- 시작 조건: 현재 플레이어의 개인 시간이 0초에 도달했거나 연결 끊김 유예 시간과 시간패 조건이 경쟁한다.
- 사용자 행동: 플레이어가 3초 유예 안에 유효 행동을 제출하거나, 서버가 종료 조건을 판정한다.
- 시스템 반응: 서버는 유예 안의 유효 행동만 인정하고, 동시에 도달한 경우 시간패를 우선한다. 경기별 직렬화 기준으로 종료 이벤트를 한 번만 저장한다.
- 사용자에게 보이는 결과: 시간패·몰수패·정상 행동 중 서버가 확정한 하나의 결과를 확인한다.
- 완료 조건: 종료 이후 행동은 상태를 변경하지 않으며 결과·전적 반영의 기준이 하나로 유지된다.

## 상태/권한별 엣지 케이스

- 경기 참가자가 아닌 사용자의 경기 상태 조회와 행동 제출은 거부한다.
- 현재 턴이 아닌 사용자의 행동은 거부한다.
- 턴 시작, 이동 전 행동, 말 이동, 프로모션, 이동 후 행동 순서를 벗어난 요청은 거부한다.
- 최초 턴 플레이어의 최초 턴에는 턴 시작 마나를 지급하지 않는다.
- 마나는 10을 초과하지 않으며 시작 마나는 양쪽 모두 0이다.
- 자신의 킹이 체크 상태일 때는 체크 해소에 기여하는 카드만 사용할 수 있다.
- 카드 사용 후 자신의 킹이 체크 상태이면 행동을 확정하지 않는다.
- 카드 효과는 상대 킹을 직접 제거하거나 표준 체스 승패·무승부 판정을 우회할 수 없다.
- 카드 효과로 만들어진 앙파상 가능 상태는 다음 적용 가능한 말 이동에만 유효하다.
- 카드 효과로 폰이 끝 행에 도달해도 프로모션은 발생하지 않는다.
- 일반 카드 덱이 비어 있으면 드로우는 실패하고 행동 슬롯을 소비하지 않는다.
- 자기 턴 동안 모든 턴 단계의 시간이 감소한다.
- 시간 0초 이후 3초 안의 유효 행동만 인정한다.
- 시간패와 몰수패가 같은 시각이면 시간패를 우선한다.
- 같은 `actionId`와 payload 재전송은 상태를 다시 변경하지 않고 기존 결과를 반환한다.
- 같은 `actionId`에 다른 payload가 오면 충돌로 거부한다.
- 서버 재시작으로 활성 경기가 유실되면 `SERVER_ABORTED` 또는 `NO_RESULT`로 무효 처리하고 전적·MMR에 반영하지 않는다.
- 서버 장애로 게임을 계속할 수 없으면 `서버 장애로 인해 게임을 진행할 수 없습니다. 게임을 종료합니다.` 문구를 표시하고 게임을 종료한다.
- 성공한 카드 사용과 체크메이트 결과는 향후 카드 밸런스 분석에 사용할 수 있는 경기 기록으로 남긴다.

## 사용자에게 보이는 결과

- 경기 시작 스냅샷과 서버 기준 상태 버전
- 턴 단계, 현재 플레이어, 합법 행동 성공·실패 결과
- 공개 보드·일반 카드 손패·마나·타이머
- 자신의 레전드 카드와 개인 오류 등 비공개 정보
- 카드 사용·드로우·체크·체크메이트·무승부·시간패·몰수패·항복 결과
- 재접속 후 최신 경기 스냅샷
- 경기 종료 사유와 승패

## 수용 기준

- 서버는 표준 체스 합법 수와 특수 수를 권위적으로 검증한다.
- 턴은 턴 시작, 이동 전 카드 행동, 말 이동, 프로모션 선택, 이동 후 행동, 턴 종료 순서를 따른다.
- 카드 사용은 카드 조건, 마나, 체크 상태, 최종 킹 안전성을 모두 통과해야 한다.
- 체크메이트·스테일메이트·기물 부족·50수·3회 반복이 판정된다.
- 이동 전 카드, 말 이동, 이동 후 카드 사용으로 체크메이트가 발생하면 즉시 종료한다.
- 시작 마나는 0이고 마나는 10을 초과하지 않는다.
- 턴 시작 마나, 캡처 마나, 앙파상 추가 마나와 체크 관련 마나가 서버에서 계산된다.
- 각 플레이어의 시간은 30분이고 자기 턴에만 감소한다.
- 시간 0초 후 3초 유예와 연결 끊김 후 20초 재접속 유예가 적용된다.
- 재접속 시 서버 권위 스냅샷으로 상태가 복구되며, 종료 결과는 되돌리지 않는다.
- 재접속한 사용자는 재접속 전과 동일한 게임 화면에서 최신 상태를 확인한다.
- 중복 행동과 동시 종료 경쟁에서 도메인 상태와 종료 이벤트가 한 번만 확정된다.
- 공개 정보와 비공개 정보가 각 권한에 맞게 분리된다.
- 서버 재시작으로 무효 처리된 활성 경기는 전적·MMR에 반영되지 않는다.
- 성공 카드 사용과 체크메이트 기물 유형은 카드·기물 통계 산출에 사용할 수 있도록 기록된다.

## 미확정 사항

- `docs/cards` 내부의 파일 구조·포맷과 카드 정보 버전 관리 방식
- MVP 카드 12장의 비용, 대상, 효과, 실패 조건과 테스트 fixture의 세부값
- 카드 효과가 표준 체스 상태에 미치는 구체적인 제품 규칙
- 카드·기물 통계의 사용자 노출 범위와 기간별 표시 방식

카드 상세 저장 위치와 서버 장애 문구, 재접속 화면 동일성은 확정되었지만, 카드 파일 구조와 세부 정책은 임의로 정하지 않는다. 체스 라이브러리, 상태 저장 방식, 명령 직렬화 방식, REST·STOMP 상세 계약은 기술 문서로 넘긴다.

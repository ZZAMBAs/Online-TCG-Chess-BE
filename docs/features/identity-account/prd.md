# Identity & Account PRD

## 목적

사용자가 서비스 계정을 안전하게 만들고 로그인한 뒤, 필수 인증·약관·닉네임 설정을 완료하여 주요 기능에 접근하도록 한다. 사용자는 자신의 계정 상태와 설정을 확인하고, 필요할 때 로그아웃·탈퇴·비밀번호 재설정을 수행할 수 있어야 한다.

## 대상 사용자

- 이메일과 비밀번호로 가입하려는 사용자
- Google 또는 Kakao OAuth로 가입·로그인하려는 사용자
- 인증·필수 설정을 완료하고 서비스를 이용 중인 사용자
- 계정 상태, 알림, 채팅 표시 설정을 확인·변경하려는 사용자

## 범위

### 포함

- 자체 이메일·비밀번호 가입과 로그인
- Google·Kakao OAuth 로그인
- 이메일 인증 토큰 발급·사용과 인증 메일 재요청
- 비밀번호 재설정 토큰을 사용하는 비밀번호 재설정
- 필수 약관 동의와 동의 시각·버전 보관
- 고유 닉네임 설정과 형식·금칙어·중복 검증
- 서버 세션과 세션 무효화
- 로그아웃, 탈퇴, 탈퇴 계정 재활성화
- `ACTIVE`, `UNVERIFIED`, `SUSPENDED`, `WITHDRAWN` 계정 상태
- 자신의 마이페이지·설정·인앱 알림 조회
- 채팅 표시, 인앱 알림, 이메일 알림 설정
- 신규 사용자 스타터 컬렉션·기본 덱 지급의 계정 온보딩 진입

### 제외

- 자체 로그인 계정과 OAuth 계정 자동 연동
- 모바일 앱·외부 공개 API용 JWT 인증
- 닉네임 변경
- 관리자 역할 부여·회수 기능
- 카드, 덱, 경기, 커뮤니티 데이터의 상세 관리

## 주요 사용자 흐름

1. 사용자가 이메일·비밀번호 또는 OAuth로 가입·로그인을 시작한다.
2. 자체 로그인 사용자는 이메일로 받은 인증 링크를 유효 시간 안에 열고, OAuth 사용자는 서비스 닉네임을 설정한다.
3. 모든 가입 방식이 필수 약관 동의를 완료하면 계정이 주요 기능 사용 조건을 충족한다.
4. 서버는 신규 계정에 스타터 컬렉션과 기본 덱을 중복 없이 지급한다.
5. 사용자는 자신의 설정과 허용된 알림을 조회·변경한다.
6. 사용자는 로그아웃하거나 탈퇴할 수 있고, 탈퇴 기록은 보존된 상태로 표시된다.

## 상세 사용자 시나리오

### 시나리오 1. 자체 로그인 사용자의 가입과 주요 기능 활성화

- 시작 조건: 가입에 사용되지 않은 이메일과 유효한 닉네임 후보가 있다.
- 사용자 행동: 이메일·비밀번호·닉네임을 제출하고 필수 약관에 동의한 뒤 이메일로 받은 인증 링크를 5분 안에 연다.
- 시스템 반응: 서버는 닉네임 형식·금칙어·중복을 검증하고, 인증 완료 후 계정을 주요 기능 사용 가능 상태로 전환한다. 신규 계정에는 스타터 컬렉션과 기본 덱을 한 번만 지급한다.
- 사용자에게 보이는 결과: 로그인된 사용자는 계정 상태가 활성화되었음을 확인하고 빠른 매치 등 주요 기능으로 이동할 수 있다.
- 완료 조건: 유효한 인증 링크로 계정 인증과 초기 지급 데이터가 중복 없이 완료되고 주요 기능 접근 조건이 충족된다.

### 시나리오 2. 이메일 미인증 상태에서 제한 기능만 이용

- 시작 조건: 자체 로그인 계정은 생성되었지만 이메일 인증이 완료되지 않았다.
- 사용자 행동: 사용자가 빠른 매치 또는 덱 기능을 요청한다.
- 시스템 반응: 서버는 계정 상태를 확인하고 주요 기능 요청을 거부한다. 이메일 인증 메일 재요청과 최소 계정 상태 조회는 허용한다.
- 사용자에게 보이는 결과: 사용자는 인증이 필요하다는 제한 안내와 인증 관련 다음 행동을 확인한다.
- 완료 조건: 매칭·덱·경기 상태가 생성되거나 변경되지 않는다.

### 시나리오 3. OAuth 가입 중 기존 자체 계정과 이메일 충돌

- 시작 조건: 같은 이메일의 활성 자체 로그인 계정이 존재한다.
- 사용자 행동: 사용자가 해당 이메일로 Google 또는 Kakao OAuth 가입을 시도한다.
- 시스템 반응: 서버는 계정을 자동 병합하지 않고 신규 OAuth 가입을 거부한다.
- 사용자에게 보이는 결과: 사용자는 기존 자체 로그인 방식으로 로그인해야 한다는 안내를 받는다.
- 완료 조건: 별도 OAuth 계정이나 자동 연결이 생성되지 않는다.

### 시나리오 4. 탈퇴와 기록 보존 후 재활성화

- 시작 조건: 활성 사용자가 로그인된 상태이며 경기·커뮤니티 기록을 보유할 수 있다.
- 사용자 행동: 사용자가 탈퇴를 확정한 뒤, 동일 이메일 또는 동일 OAuth 식별자로 다시 가입을 시도한다.
- 시스템 반응: 서버는 계정을 물리 삭제하지 않고 `WITHDRAWN`으로 전환하며 기존 기록의 표시명을 탈퇴 사용자로 처리한다. 재가입 시 기존 비활성 계정임을 검증한 뒤 기존 닉네임을 유지하여 재활성화한다.
- 사용자에게 보이는 결과: 탈퇴 직후에는 로그인과 알림을 사용할 수 없고, 재활성화 후에는 기존 계정으로 다시 접근한다.
- 완료 조건: 기록 무결성이 유지되고, 중복 계정이 생성되지 않는다.

### 시나리오 5. 계정 설정 변경과 민감 필드 보호

- 시작 조건: 사용자가 활성 계정으로 자신의 설정 화면에 접근한다.
- 사용자 행동: 채팅 표시 여부, 인앱 알림 수신 여부 또는 이메일 알림 수신 여부를 변경하고, 임의 `userId` 또는 `role` 같은 필드를 함께 제출한다.
- 시스템 반응: 서버는 현재 세션 사용자만 대상으로 허용된 설정만 변경하고 알 수 없는 필드와 민감 필드 변경은 거부한다. 채팅 표시 설정은 전송 권한·저장·신고·차단 정책을 바꾸지 않는다.
- 사용자에게 보이는 결과: 허용된 설정 변경 결과를 확인하고, 허용되지 않은 필드는 변경되지 않았음을 확인한다.
- 완료 조건: 다른 사용자 정보와 역할·계정 상태·이메일·닉네임·MMR은 변경되지 않는다.

### 시나리오 6. 비밀번호 재설정 토큰 사용과 마이페이지 요약 조회

- 시작 조건: 사용자가 비밀번호 재설정을 요청할 수 있는 계정 식별 정보를 가지고 있다.
- 사용자 행동: 사용자가 비밀번호 재설정을 요청하고, 메일로 받은 재설정 토큰을 5분 안에 사용해 비밀번호를 변경한 뒤 마이페이지 첫 화면을 조회한다.
- 시스템 반응: 서버는 해시된 재설정 토큰의 유효성과 만료를 검증하고 1회 사용 후 무효화한다. 마이페이지 첫 화면에는 간단한 사용자 프로필과 승·무·패 및 승률 수준의 전적 요약만 제공한다.
- 사용자에게 보이는 결과: 사용자는 토큰을 통해 비밀번호를 변경하고, 마이페이지에서 상세 정보가 아닌 기본 프로필과 요약 전적을 확인한다.
- 완료 조건: 비밀번호 재설정 요청에서 이메일 존재 여부가 노출되지 않고, 마이페이지 첫 화면에 상세 전적·기보가 포함되지 않는다.

## 상태/권한별 엣지 케이스

- `UNVERIFIED` 사용자는 로그인·로그아웃·인증 메일 재요청·비밀번호 재설정·최소 계정 상태 조회만 허용된다.
- `SUSPENDED` 사용자는 로그인 후 제한 안내·제재 사유/기간 확인·로그아웃만 허용된다.
- `WITHDRAWN` 사용자는 로그인하거나 알림을 받을 수 없다.
- `ADMIN` 역할은 일반 사용자 기능에 추가된 관리자 기능을 사용할 수 있지만, 계정 상태 제한은 역할보다 우선한다.
- 같은 닉네임은 대소문자를 구분하지 않고 중복될 수 없으며, 2~16자의 한글·영문·숫자·언더스코어만 허용한다.
- 이메일 인증 토큰과 비밀번호 재설정 토큰은 해시로 저장되고 모두 5분 동안 유효하며 1회 사용 후 무효화된다.
- 새 인증·재설정 토큰을 발급하면 같은 사용자의 기존 미사용 토큰은 무효화된다.
- 비밀번호 재설정 요청은 이메일 존재 여부를 노출하지 않는다.
- 로그인 5회 연속 실패 시 10분 잠금이 적용되며, 비밀번호 재설정 완료 시 실패 횟수와 잠금이 초기화된다.
- 로그아웃·탈퇴·계정정지·비밀번호 재설정 완료 후 기존 세션과 STOMP 연결은 무효화된다.
- 마이페이지와 설정은 세션 사용자의 데이터만 대상으로 하며 임의 `userId`를 받지 않는다.
- 계정정지 사용자의 주요 마이페이지·설정 접근은 제한된다.

## 사용자에게 보이는 결과

- 가입·로그인·OAuth 충돌·인증 필요·계정정지·탈퇴 상태를 구분할 수 있는 상태 안내
- 인증 메일 재요청과 비밀번호 재설정 진행 결과
- 자신의 계정 상태, 닉네임, 설정, 허용된 인앱 알림
- 마이페이지 첫 화면의 간단한 사용자 프로필과 승·무·패 및 승률 요약
- 신규 계정의 스타터 컬렉션과 기본 덱 지급 결과
- 설정 변경 성공 결과와 허용되지 않은 필드 거부 결과
- 탈퇴 후 로그인 불가 및 재활성화 결과

## 수용 기준

- 이메일·비밀번호 가입과 Google·Kakao OAuth 로그인이 가능하다.
- 이메일 인증 전 자체 로그인 사용자는 주요 기능을 사용할 수 없다.
- OAuth 사용자는 닉네임 설정과 필수 약관 동의 전 주요 기능을 사용할 수 없다.
- 동일 이메일의 활성 자체 로그인 계정이 있으면 OAuth 신규 가입은 자동 병합되지 않는다.
- 약관 동의 시각과 버전이 가입 방식과 무관하게 보관된다.
- 닉네임 형식·금칙어·대소문자 비구분 중복 검증이 적용된다.
- 이메일 인증 토큰과 비밀번호 재설정 토큰은 5분 동안 유효하고 1회 사용 후 무효화된다.
- 세션 쿠키는 `HttpOnly`, `Secure`, `SameSite=Lax` 기준을 따른다.
- REST 상태 변경과 STOMP 연결 시 인증·CSRF·Origin·계정 상태·제재가 검증된다.
- 신규 사용자에게 스타터 컬렉션과 기본 덱이 멱등적으로 지급된다.
- `UNVERIFIED`, `SUSPENDED`, `WITHDRAWN`별 허용·거부 기능을 검증할 수 있다.
- 탈퇴 사용자는 로그인할 수 없고 기존 기록은 탈퇴 사용자로 표시된다.
- 마이페이지와 설정은 세션 사용자 자신만 조회·수정할 수 있다.
- 마이페이지 첫 화면은 간단한 프로필과 승·무·패 및 승률 요약을 제공한다.
- 민감 필드와 알 수 없는 설정 필드는 변경할 수 없다.
- 로그아웃·탈퇴·계정정지·비밀번호 재설정 완료 후 기존 세션과 STOMP 연결이 무효화된다.

## 미확정 사항

- 이메일 인증·비밀번호 재설정 메일의 사용자 화면 문구와 SMTP 제공자 세부
- OAuth 제공자별 실패·취소 화면의 제품 문구
- 탈퇴 계정 재활성화 시 추가 검증 절차의 구체적 사용자 흐름

위 항목은 원천 명세에 없는 제품 상세이므로 임의 확정하지 않는다. 세션 저장소, 보안 필터 구성, endpoint와 payload는 기술 문서로 넘긴다.

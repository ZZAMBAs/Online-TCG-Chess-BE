# Match Chat PRD

## 목적

경기 참가자 두 명이 경기 중 안전한 텍스트 채팅을 주고받고, 차단·제재·금칙어 정책이 적용된 결과를 확인하도록 한다.

## 대상 사용자

- 진행 중인 경기에 참가한 두 플레이어
- 경기 채팅을 차단하거나 신고하려는 사용자
- 신고된 채팅의 원문을 검토하는 관리자

## 범위

### 포함

- 경기 참가자 전용 채팅 전송·구독
- 서버 세션 기준 발신자 식별
- plain text, 빈 값·공백·길이 검증
- 사용자+경기 기준 Rate Limit
- 금칙어 마스킹과 원문·마스킹 결과 보관
- 수신자별 차단 전달 제한
- 경기 종료 후 전송 차단
- 채팅 신고와 신고 증거 스냅샷
- 관리자 권한 기반 원문 조회와 감사 대상 처리
- 일반 채팅 로그 7일 보관

### 제외

- 경기 외 1:1 메시지
- 이미지·HTML·리치 텍스트·파일 전송
- 커뮤니티 댓글 채팅
- 금칙어 우회 탐지와 외부 필터 서비스
- 제재·차단 관리 화면의 상세 흐름

## 주요 사용자 흐름

1. 경기 참가자가 경기 채널에 연결하고 채팅 입력을 전송한다.
2. 서버가 인증 세션, 경기 참가 여부, 계정 상태, 제재, 메시지 형식과 Rate Limit을 검증한다.
3. 서버가 금칙어를 마스킹한 메시지를 수신자별 채널로 전달한다.
4. 차단 관계가 있으면 차단자에게 해당 메시지를 전달하지 않는다.
5. 사용자가 문제 메시지를 신고하면 서버가 원문·마스킹 결과와 증거를 보관한다.
6. 관리자는 권한과 감사 로그 조건을 충족할 때만 원문을 검토한다.

## 상세 사용자 시나리오

### 시나리오 1. 정상 채팅 전송과 금칙어 마스킹

- 시작 조건: 두 사용자가 진행 중인 경기에 참가하고 있고 채팅 제재가 없다.
- 사용자 행동: 한 플레이어가 100자 이하의 plain text 메시지를 전송한다.
- 시스템 반응: 서버는 발신자를 세션에서 확인하고 금칙어를 대소문자 비구분 단순 부분 문자열 기준으로 찾아 전체 길이만큼 `*`로 치환한다. 원문과 마스킹 결과를 저장한다.
- 사용자에게 보이는 결과: 수신자는 HTML이 실행되지 않는 마스킹 메시지를 받는다.
- 완료 조건: 클라이언트가 보낸 senderId·닉네임·권한은 신뢰되지 않고, 사용자 화면에는 원문이 노출되지 않는다.

### 시나리오 2. 차단한 상대의 채팅

- 시작 조건: 사용자가 상대를 차단했으며 두 사용자가 같은 경기에 참가 중이다.
- 사용자 행동: 차단된 상대가 채팅을 전송한다.
- 시스템 반응: 서버는 메시지를 정책에 따라 저장하고 신고 증거 대상이 될 수 있게 처리하되, 차단자 개인 채널에는 전달하지 않는다. 차단 여부는 송신자에게 알리지 않는다.
- 사용자에게 보이는 결과: 차단자는 해당 메시지를 받지 않고, 송신자는 일반적인 전송 결과만 확인한다.
- 완료 조건: 차단이 채팅 저장·신고 증거 정책을 우회하지 않으며 차단 관계가 상대에게 노출되지 않는다.

### 시나리오 3. 제재·종료·Rate Limit 거부

- 시작 조건: 사용자가 커뮤니케이션 정지 상태이거나 경기 종료 후이거나 짧은 시간에 전송 한도를 초과했다.
- 사용자 행동: 사용자가 채팅 전송을 요청한다.
- 시스템 반응: 서버는 요청을 거부하고 적절한 제한 결과를 반환한다. 경기 종료 후 메시지나 제한 초과 메시지는 채팅 상태를 변경하지 않는다.
- 사용자에게 보이는 결과: 사용자는 메시지가 전달되지 않았고 제한 사유를 확인한다.
- 완료 조건: 금지된 메시지가 상대에게 전달되지 않고, 실패 요청이 정상 채팅 로그로 저장되지 않는다.

### 시나리오 4. 채팅 신고와 관리자 원문 검토

- 시작 조건: 사용자가 경기 채팅 메시지를 확인했고 해당 메시지를 신고할 수 있다.
- 사용자 행동: 사용자가 `chatMessageId`를 기준으로 신고하고, 관리자가 신고 증거를 검토한다.
- 시스템 반응: 서버는 원문·마스킹 결과·메시지·경기·발신자·신고자 정보를 증거 스냅샷으로 보관한다. 관리자는 권한 검증과 감사 기록 후 원문을 조회한다.
- 사용자에게 보이는 결과: 일반 사용자는 마스킹 결과만 보고, 권한 있는 관리자는 검토 과정에서 원문을 확인한다.
- 완료 조건: 관리자 원문 조회가 감사 로그로 남고 일반 사용자에게 원문이 노출되지 않는다.

## 상태/권한별 엣지 케이스

- 경기 참가자가 아닌 사용자의 채팅 구독·전송은 거부한다.
- 서버는 클라이언트의 senderId, 닉네임, 권한을 신뢰하지 않는다.
- `UNVERIFIED`, `SUSPENDED`, `WITHDRAWN` 및 커뮤니케이션 정지 사용자는 정책에 따라 채팅 전송이 제한된다.
- 계정정지 사용자는 채팅을 포함한 주요 기능을 사용할 수 없다.
- 경기 종료 후 채팅 전송은 거부한다.
- 빈 문자열, 공백만 있는 메시지, 100자 초과 메시지는 거부한다.
- HTML 등 리치 콘텐츠는 허용하지 않고 화면과 관리자 화면 모두 escape한다.
- 사용자+경기 기준 1초 3건, 10초 10건을 초과하면 제한한다.
- 금칙어는 대소문자 구분 없이 단순 부분 문자열로 검사하고 중첩 시 긴 단어를 우선한다.
- 공백·특수문자 우회, 초성 분리, Unicode confusable 탐지는 MVP에서 제공하지 않는다.
- 금칙어 전체 길이만큼 `*`로 마스킹한다.
- 차단자에게 메시지를 전달하지 않지만 저장·신고 증거 정책은 유지한다.
- 같은 사용자의 중복 차단·신고 요청은 멱등 정책을 따른다.
- 일반 채팅 로그는 7일 보관하고 신고 증거는 별도 보관 정책을 따른다.

## 사용자에게 보이는 결과

- 경기 참가자에게만 제공되는 채팅 입력·수신 결과
- 금칙어가 마스킹된 plain text 메시지
- 빈 값·길이 초과·제재·경기 종료·Rate Limit 거부 안내
- 차단된 상대의 메시지가 표시되지 않는 결과
- 채팅 신고 접수 결과
- 일반 사용자에게 숨겨진 관리자 원문 검토

## 수용 기준

- 경기 참가자만 채팅을 구독·전송할 수 있다.
- 서버는 인증 세션 기준으로 발신자를 확정한다.
- plain text만 허용하고 빈 값·공백·100자 초과 메시지를 거부한다.
- HTML escape와 금칙어 마스킹이 적용된다.
- 금칙어는 대소문자 구분 없는 단순 부분 문자열 기준으로 처리한다.
- 사용자+경기 기준 1초 3건, 10초 10건 Rate Limit이 적용된다.
- 커뮤니케이션 정지·계정정지·경기 종료 상태에서 전송할 수 없다.
- 차단한 사용자의 메시지는 차단자에게 전달되지 않는다.
- 채팅 원문과 마스킹 결과가 저장되고 일반 로그는 7일 보관된다.
- 채팅 신고는 메시지 ID와 신고 증거 스냅샷을 기준으로 저장된다.
- 관리자 원문 조회는 권한 검증과 감사 로그를 거친다.

## 미확정 사항

- 신고 증거 스냅샷의 정확한 보관 만료 처리 문구와 사용자 안내
- 금칙어 관리자가 사전을 변경했을 때 기존 메시지의 재마스킹 여부
- 채팅 연결 해제·재연결 시 최근 메시지 표시 범위

금칙어 우회 탐지와 외부 필터 연동은 MVP 이후 확장 사항이다. STOMP destination, payload, 저장 schema와 Rate Limit 구현 방식은 기술 문서로 넘긴다.

# Match History PRD

## 목적

종료된 경기의 결과와 기보를 서버 기준으로 보존하고, 참가자가 자신의 전적·최근 경기·기보를 안전하게 조회할 수 있도록 한다. 종료 경기 기록은 카드와 기물의 밸런스 분석 통계에도 사용할 수 있어야 하며, 진행 중이거나 무효인 경기는 전적에 섞이지 않아야 한다.

## 대상 사용자

- 종료된 자신의 경기 결과와 전적을 확인하려는 사용자
- 자신이 참가한 경기의 상세 기보를 다시 확인하려는 사용자

## 범위

### 포함

- 승리·패배·무승부 수와 승률
- 종료 경기 수와 최근 경기 목록
- 상대 닉네임, 색, 시작·종료 시각, 종료 사유, 결과
- 경기 종료 시 결과·전적·MMR·기보 봉인의 일관된 확정
- 서버가 확정한 순서와 결과에 따른 기보 보존
- 경기 참가자 본인의 기보 조회
- 조회자에 따른 공개·비공개 정보 구분
- 탈퇴 사용자 표시와 과거 기보 권한 유지
- 서버 장애 무효 경기의 전적·MMR 제외
- 카드별 경기 투입·승리와 체크메이트 기물 통계를 위한 종료 경기 집계 기준

### 제외

- 진행 중·취소·미시작 경기의 전적 반영
- 공개 기보와 제3자 기보 조회
- 관리자 기보 조회 권한
- 전적·기보의 수정 또는 삭제
- 상세 경기 진행과 카드 효과 계산

## 주요 사용자 흐름

1. 경기가 체크메이트, 무승부, 시간패, 몰수패 또는 항복으로 종료된다.
2. 서버가 종료 사유와 시각을 확정한다.
3. 서버가 결과, 전적, MMR과 기보 봉인을 함께 확정한다.
4. 사용자가 마이페이지에서 자신의 전적 요약과 최근 경기 목록을 조회한다.
5. 사용자가 자신이 참가한 경기의 기보를 조회한다.
6. 서버가 조회자에게 허용된 공개 정보만 제공한다.

## 상세 사용자 시나리오

### 시나리오 1. 정상 경기 종료와 전적 반영

- 시작 조건: 두 사용자가 진행 중인 경기에 참가하고 있다.
- 사용자 행동: 경기 종료 조건이 확정되도록 플레이를 완료한다.
- 시스템 반응: 서버는 종료 사유·결과·종료 시각, 전적, MMR, 기보 봉인과 카드·기물 통계에 필요한 종료 경기 기록을 하나의 결과로 처리한다.
- 사용자에게 보이는 결과: 사용자는 경기 결과와 갱신된 승·무·패·승률 및 최근 경기 항목을 확인한다.
- 완료 조건: 같은 종료가 중복 반영되지 않고 종료된 경기만 전적에 포함된다.

### 시나리오 2. 자신의 경기 기보 조회

- 시작 조건: 사용자가 종료된 경기의 참가자다.
- 사용자 행동: 사용자가 자신의 과거 경기 기보를 조회한다.
- 시스템 반응: 서버는 참가자 권한을 확인하고 당시 확정된 행동과 결과를 턴별·행동 단계별 기보로 구성한다. 발동되어 공개된 카드와 효과 성공·실패는 표시하되, 사용되지 않은 상대 레전드와 카드 대상 후보 같은 비공개 정보는 제외한다.
- 사용자에게 보이는 결과: 사용자는 턴별로 구분된 기보에서 카드 사용·패스·드로우·기물 이동·마나 변화와 종료 이벤트를 확인하고, 승·무·패가 결정된 경우 승자를 확인한다.
- 완료 조건: 기보는 서버가 봉인한 기록과 일치하고 숨김 정보가 노출되지 않는다.

### 시나리오 3. 비참가자의 기보 접근 거부

- 시작 조건: 사용자가 해당 경기에 참가하지 않았거나 다른 사용자의 기보를 요청한다.
- 사용자 행동: 사용자가 경기 기보를 조회한다.
- 시스템 반응: 서버는 경기 참가자 여부를 확인하고 접근을 거부한다. 공개 기보 기능은 MVP에서 제공하지 않는다.
- 사용자에게 보이는 결과: 사용자는 권한 없음 안내를 받고 기보 원문을 확인할 수 없다.
- 완료 조건: 상대의 카드·기보·개인 정보가 응답에 포함되지 않는다.

### 시나리오 4. 서버 장애로 무효 처리된 경기

- 시작 조건: 서버 재시작 또는 장애로 활성 경기 상태가 유실되었다.
- 사용자 행동: 사용자가 해당 경기의 결과와 전적을 조회한다.
- 시스템 반응: 서버는 무효 처리 직전까지 확정된 기보를 보존하고 경기를 무효로 처리한다. 해당 경기는 전적·MMR에 반영하지 않는다.
- 사용자에게 보이는 결과: 사용자는 무효 처리 전까지의 기보를 확인하고, 마지막 승자 표시 위치에서 정상 승패가 아닌 무효 경기임을 확인한다.
- 완료 조건: 무효 경기의 결과가 승·무·패와 승률에 영향을 주지 않으며, 무효 처리 전 기록은 보존된다.

### 시나리오 5. 탈퇴 사용자와 과거 기록

- 시작 조건: 과거 경기 참가자 중 한 명이 탈퇴했다.
- 사용자 행동: 다른 참가자가 과거 경기 목록이나 기보를 조회한다.
- 시스템 반응: 서버는 과거 기록의 무결성을 유지하면서 탈퇴 사용자를 탈퇴 사용자로 표시한다. 참가자의 과거 기보 권한은 차단 관계와 무관하게 유지한다.
- 사용자에게 보이는 결과: 상대의 기존 닉네임 대신 탈퇴 사용자 표시를 확인하고 기보를 계속 조회한다.
- 완료 조건: 탈퇴·차단 상태가 기보 응답에 불필요하게 노출되지 않는다.

## 상태/권한별 엣지 케이스

- 진행 중, 취소, 미시작, 무효 경기는 전적에 포함하지 않는다.
- 서버가 최종 확정한 경기만 결과·전적·MMR에 반영한다.
- 동일 경기의 종료 조건이 겹쳐도 결과는 한 번만 확정된다.
- 시간패·몰수패·항복은 확정된 종료 사유로 보존한다.
- 서버 장애로 무효가 된 경기는 전적과 MMR에서 제외한다.
- 최근 경기 목록은 종료 시각 내림차순으로 정렬하고 한 번에 최대 10개씩 무한 스크롤로 조회한다.
- 경기 참가자만 자신의 기보를 조회할 수 있다.
- 사용되지 않은 상대 레전드는 경기 종료 후에도 기보에 공개하지 않는다.
- 발동된 카드는 효과가 실패했더라도 공개된 사용 기록과 비용 소비 결과를 기보에 남긴다.
- 대상 선택 후보 전체와 상대에게 공개할 필요가 없는 선택 세부 정보는 기보에 노출하지 않는다.
- 과거 기보는 당시 경기에 고정된 카드 식별자·버전과 당시 확정된 결과를 따르며, 현재 카드 효과로 다시 해석하지 않는다.
- 난수 생성의 비공개 정보, 운영 기록, 비공개 MMR, 접속 정보와 차단 관계는 기보에 포함하지 않는다.
- 탈퇴 사용자는 기록에서 탈퇴 사용자로 표시한다.
- 과거 경기 기보 권한은 차단 관계만으로 제거하지 않는다.

## 사용자에게 보이는 결과

- 총 종료 경기 수, 승리·패배·무승부 수, 승률
- 최근 경기의 상대, 색, 시작·종료 시각, 종료 사유, 결과
- 정상 종료와 무효 종료의 구분
- 자신의 경기에서 조회 가능한 상세 기보
- 탈퇴 사용자 표시와 숨김 정보가 제외된 기보
- 비참가자 접근 거부 안내

## 수용 기준

- 종료된 경기만 전적에 반영된다.
- 승률은 승리 수를 종료 경기 수로 나누고 무승부를 분모에 포함하며, 소수점 첫째자리까지 표시하고 둘째자리에서 반올림한다.
- 최근 경기 목록은 종료 시각 내림차순으로 최대 10개씩 무한 스크롤한다.
- 경기 종료 시 결과, 전적, MMR과 기보 봉인이 함께 확정된다.
- 기보는 서버가 당시 확정한 행동 순서와 결과를 보존한다.
- 경기 참가자만 자신의 기보를 조회할 수 있다.
- 사용되지 않은 상대 레전드와 비공개 운영 정보가 기보에 노출되지 않는다.
- 발동된 카드의 공개·비용 소비·효과 성공 또는 실패를 당시 카드 버전과 함께 확인할 수 있다.
- 과거 기보는 현재 카드 정의로 효과를 다시 계산하지 않는다.
- 서버 장애 무효 경기는 전적·MMR에 반영되지 않는다.
- 종료 경기 기록은 카드별 경기 투입 수·승리 수와 체크메이트 기물 통계 산출에 사용할 수 있다.
- 탈퇴 사용자의 과거 기록은 탈퇴 사용자로 표시된다.
- 동일 경기 종료가 중복으로 전적에 반영되지 않는다.

## 미확정 사항

- 최근 경기 무한 스크롤의 추가 정렬·필터 조건
- 특별한 행동이나 카드 효과를 기보에 표시하는 세부 문구와 표현
- 무효 경기 표시의 시각적 세부 표현
- 카드별 통계의 사용자 노출 범위, 집계 기간과 카드 버전별 해석

최근 경기 10개 단위 무한 스크롤, 승률 반올림, 턴·행동 단계별 기보, 무효 전까지의 기록 보존과 당시 결과 기반 재현은 확정되었다. 조회 형식과 기록 저장 방식은 후속 계약·기술 문서에서 다룬다.

# Matchmaking PRD

## 목적

인증과 필수 설정을 완료하고 유효한 덱을 가진 사용자가 빠른 매치 대기열에 참여하고, 서버가 조건에 맞는 상대를 찾아 경기 생성까지 일관되게 확정하도록 한다.

## 대상 사용자

- 유효한 덱으로 1:1 온라인 경기를 시작하려는 인증 사용자
- 매칭 대기 상태를 확인하거나 매칭 전 취소하려는 사용자

## 범위

### 포함

- 빠른 매치 진입·상태 조회·취소
- 사용자당 활성 대기열 항목 하나만 허용
- 유효한 덱과 계정 상태·제재 검증
- 비공개 MMR 기반 상대 탐색
- 대기 시간에 따른 허용 MMR 범위 확대
- 신규 사용자의 최초 MMR 1400
- 표준 Elo 기대 승률 공식과 `K=32`에 따른 비공개 MMR 갱신
- 차단 관계 검증
- 매칭 확정, 백·흑 무작위 배정, 경기 생성
- 경기 생성 시 덱 스냅샷과 초기 상태 확정
- 매칭·경기 생성 결과의 중복 방지

### 제외

- 친선 코드, 랭크 모드, 관전
- 체스 수와 카드 행동 등 경기 진행
- 경기 중 타이머·재접속·항복·종료 처리
- 공개 MMR, 티어, 시즌, 리더보드
- 분산 매칭 큐와 다중 인스턴스 매칭

## 주요 사용자 흐름

1. 사용자가 유효한 덱으로 빠른 매치 진입을 요청한다.
2. 서버가 인증, 계정 상태, 제재, 덱 유효성, 중복 대기 여부를 검증한다.
3. 서버가 대기열에 사용자를 등록하고 대기 상태를 제공한다.
4. 서버가 비공개 MMR과 대기 시간, 차단 관계를 기준으로 상대 후보를 찾는다.
5. 매칭 확정 직전에 양쪽 조건을 다시 검증한다.
6. 서버가 백·흑, 덱 스냅샷, 초기 경기 상태를 확정하고 경기를 생성한다.
7. 양쪽 사용자에게 매칭 확정과 경기 시작 결과를 제공한다.

## 상세 사용자 시나리오

### 시나리오 1. 유효한 덱으로 빠른 매치 대기열 진입

- 시작 조건: 사용자가 주요 기능을 사용할 수 있는 계정 상태이고 유효한 덱을 보유한다.
- 사용자 행동: 사용자가 빠른 매치 진입을 요청한다.
- 시스템 반응: 서버는 계정 상태, 기능 제재, 덱 유효성, 활성 대기열 중복 여부를 검증한 뒤 대기열에 한 건을 등록한다.
- 사용자에게 보이는 결과: 사용자는 `WAITING` 상태와 현재 대기 상태를 확인한다.
- 완료 조건: 사용자의 활성 대기열 항목은 하나만 존재한다.

### 시나리오 2. 대기 중 취소

- 시작 조건: 사용자의 빠른 매치 상태가 `WAITING`이다.
- 사용자 행동: 사용자가 매칭 취소를 요청한다.
- 시스템 반응: 서버는 아직 매칭이 확정되지 않았는지 확인하고 대기열 항목을 `CANCELED`로 전환한다.
- 사용자에게 보이는 결과: 사용자는 대기 취소 결과를 확인하고 새로운 매칭을 시작할 수 있다.
- 완료 조건: 취소된 대기열 항목으로 경기가 생성되지 않는다.

### 시나리오 3. 상대 매칭과 원자적 경기 생성

- 시작 조건: 서로 매칭 가능한 두 사용자가 대기 중이고 차단 관계가 없다.
- 사용자 행동: 별도 사용자 행동 없이 서버가 매칭을 확정한다.
- 시스템 반응: 서버는 매칭 확정 직전에 계정 상태, 제재, 덱 유효성, 차단 관계를 재검증한다. 이후 백·흑을 무작위 배정하고 각 덱을 스냅샷으로 고정한 경기와 초기 상태를 생성한다.
- 사용자에게 보이는 결과: 양쪽 사용자는 매칭 확정, 자신의 색, 경기 시작과 초기 상태를 확인한다.
- 완료 조건: 두 사용자가 같은 경기와 고정된 덱 스냅샷을 공유하며, 경기 생성이 중복되지 않는다.

### 시나리오 4. 매칭 확정 직전 조건 불충족

- 시작 조건: 사용자가 대기열에 들어간 뒤 덱을 변경했거나 상대와 새 차단 관계가 생겼다.
- 사용자 행동: 서버가 해당 후보를 매칭 확정하려 한다.
- 시스템 반응: 서버는 최종 검증에서 조건 불충족을 확인하고 경기 생성을 진행하지 않는다. 대기열 또는 매칭 상태는 실패 또는 재탐색 가능한 상태로 처리한다.
- 사용자에게 보이는 결과: 사용자는 경기 시작이 거부되었거나 매칭이 다시 진행된다는 결과를 확인한다.
- 완료 조건: 유효하지 않은 덱 또는 차단 관계가 있는 상대와 경기가 생성되지 않는다.

### 시나리오 5. 중복 진입과 확정 후 취소 경쟁

- 시작 조건: 사용자가 이미 대기 중이거나 매칭 확정 상태에 있다.
- 사용자 행동: 사용자가 다시 진입하거나 취소를 요청한다.
- 시스템 반응: 대기 중 중복 진입에는 현재 상태를 반환하고, 매칭 확정 후 취소에는 이미 확정된 경기 상태를 우선한다.
- 사용자에게 보이는 결과: 중복 경기나 중복 대기열이 생성되지 않으며, 확정 후 취소가 경기 생성을 되돌리지 않는다.
- 완료 조건: 대기열과 경기 상태가 한 번만 확정된다.

## 상태/권한별 엣지 케이스

- `UNVERIFIED`, `SUSPENDED`, `WITHDRAWN` 사용자는 빠른 매치에 진입할 수 없다.
- 커뮤니케이션 정지는 매칭과 경기 진행 자체를 막지 않지만, 그 외 계정정지 등 보호 제재는 매칭을 제한한다.
- 유효하지 않은 덱으로 진입할 수 없으며, 대기 후 경기 생성 직전 덱이 무효가 되어도 경기를 만들지 않는다.
- 이미 대기 중인 사용자의 재진입은 중복 등록이 아니라 현재 상태 반환으로 처리한다.
- 매칭 확정 전 취소와 확정 후 취소의 결과가 다르다.
- 후보 탐색 시점과 매칭 확정 직전에 차단 관계를 모두 확인한다.
- 대기 시간이 길어져도 차단 관계를 무시하지 않는다.
- 서버 장애 또는 최종 검증 실패로 경기 생성이 불가능하면 매칭을 `FAILED`로 처리할 수 있다.
- 서버가 배정한 색은 무작위이며 최근 색 이력 보정은 적용하지 않는다.
- 경기 생성 후 덱을 변경해도 이미 생성된 경기의 덱 스냅샷은 변하지 않는다.

## 사용자에게 보이는 결과

- 빠른 매치 진입 성공·거부 사유
- `WAITING`, `CANCELED`, `MATCH_CONFIRMED`, `GAME_CREATED`, `FAILED` 상태
- 매칭 취소 결과
- 매칭 확정 결과와 자신의 백·흑 색
- 경기 시작과 초기 상태
- 중복 진입, 확정 후 취소, 덱 무효, 차단 관계로 인한 거부 결과

## 수용 기준

- 유효한 덱을 가진 사용자는 빠른 매치에 진입할 수 있다.
- 유효하지 않은 덱, 제한 계정, 적절한 권한이 없는 요청은 거부된다.
- 한 사용자는 활성 대기열 항목을 하나만 가진다.
- 중복 진입은 중복 등록 없이 현재 상태를 반환한다.
- 매칭 확정 전 취소가 가능하다.
- 매칭 확정 후 취소보다 이미 생성된 경기 상태가 우선한다.
- MMR이 가까운 상대를 우선하고 대기 시간에 따라 허용 범위를 넓힌다.
- 신규 사용자의 최초 MMR은 1400이다.
- 표준 Elo 기대 승률 공식과 `K=32`를 사용하며 승리·패배·무승부는 각각 1·0·0.5로 계산한다.
- 시간패와 몰수패는 패배로 반영하고 서버 장애 무효 경기는 MMR에 반영하지 않는다.
- 매칭 시작 후 0~10초는 ±50, 10~20초는 ±100, 20~30초는 ±200 범위로 상대를 찾는다.
- 차단 관계가 있는 사용자끼리는 후보 탐색과 확정 시점 모두 매칭되지 않는다.
- 매칭 확정, 경기 생성, 색 배정, 덱 스냅샷, 초기 상태 생성이 함께 확정된다.
- 서버는 백·흑을 무작위 배정하고 백이 먼저 행동한다.
- 경기 생성 후 덱 변경이 진행 중 경기에 영향을 주지 않는다.
- 중복 요청과 최종 검증 실패 상황에서 중복 경기나 잘못된 경기가 생성되지 않는다.

## 미확정 사항

- 매칭 범위 두 배 확대 정책의 장기 상한과 운영 중 조정 기준
- 대기열 실패 시 사용자에게 제공하는 재시도·재탐색 화면 문구

현재 확정값은 단순한 MVP 정책이며 추후 요구사항 변경으로 조정될 수 있다. 대기열 저장 방식, 동시성 처리, endpoint와 payload는 기술 문서로 넘긴다.

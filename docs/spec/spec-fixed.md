# TCG Online Chess Backend MVP 요구사항 명세

## 1. 개요

TCG Online Chess는 표준 체스 규칙을 기반으로, 경기 중 카드 효과를 제한적으로 결합한 실시간 1:1 온라인 웹 게임이다. 본 문서는 백엔드 MVP 구현을 위한 요구사항 명세이며, React 프론트엔드와 Spring Boot 백엔드, STOMP WebSocket 기반 실시간 통신을 전제로 한다.

기본 언어는 한국어이며, 서비스 운영 기준 시간대는 KST(Asia/Seoul)로 한다.

  - 자체 로그인 사용자의 가입 시 이메일 인증

- 서버 세션 기반 인증
- 실시간 PvP 빠른 매치
- 표준 체스 기반 게임 룰
- 일반 카드 및 레전드 카드 시스템
- 마나 시스템
- 대전 중 상대 채팅
- 카드팩 구매 및 개봉
- 카드 보유 컬렉션
- 덱 편성

  - 전적 및 기보 조회

다음 기능은 MVP에서 제외하고 이후 확장 대상으로 둔다.

- 솔로플레이 봇
- 실시간 관전

- 모바일 앱 또는 외부 공개 API용 JWT 인증

## 3. 인증 및 계정

### 3.1 인증 방식

MVP의 웹 인증은 서버 세션과 쿠키를 사용한다.

- 세션 쿠키는 `HttpOnly`, `Secure`, `SameSite` 속성을 사용한다.
- 프론트엔드와 백엔드는 같은 사이트 배포를 전제로 한다.
- REST의 상태 변경 요청은 CSRF 검증을 적용한다.
- STOMP WebSocket `CONNECT` 요청도 세션과 CSRF를 검증한다.
- WebSocket 연결이 성공하면 서버는 STOMP 세션에 인증 사용자 식별자를 연결한다.

JWT Access/Refresh Token 방식은 MVP에서 사용하지 않는다. 향후 모바일 앱, 외부 API, 멀티 클라이언트 지원이 필요해질 때 별도 인증 방식으로 추가 검토한다.

### 3.2 회원가입 및 로그인

- 자체 로그인은 이메일, 비밀번호, 고유 닉네임을 사용한다.
- 닉네임은 서비스 내 공개 식별자이며 가입 후 변경할 수 없다.
- 자체 로그인 사용자는 이메일 인증을 완료해야 매칭, 커뮤니티 작성, 카드팩 개봉 등 주요 기능을 사용할 수 있다.
- OAuth 제공자는 Google, Kakao를 지원한다.
- OAuth 계정도 서비스 고유 닉네임을 설정해야 한다.

### 3.3 비밀번호 및 로그인 보안

- 비밀번호는 최소 8자 이상이어야 하며, 최대 길이 제한을 둔다.
- 비밀번호는 안전한 해시 알고리즘으로 저장한다.
- 로그인 실패 횟수 제한을 적용한다.
- 짧은 기간의 계정 잠금을 지원한다.
- 비밀번호 재설정 토큰은 만료 시간을 가진다.

### 3.4 계정 탈퇴

- 사용자는 계정 탈퇴를 요청할 수 있다.
- MVP에서는 계정을 물리 삭제하지 않고 비활성화한다.
- 전적, 기보, 커뮤니티 기록은 서비스 무결성을 위해 보존하되, 작성자 표시는 탈퇴 사용자로 처리한다.

## 4. 실시간 통신

### 4.1 WebSocket 방식

- Spring WebSocket + STOMP를 사용한다.
- 클라이언트는 행동 의도만 서버에 전송한다.
- 서버는 모든 게임 상태, 합법 수, 카드 조건, 타이머, 승패를 권위적으로 계산한다.
- 클라이언트가 보낸 상태 계산 결과는 신뢰하지 않는다.

### 4.2 주요 실시간 이벤트

서버는 최소한 다음 유형의 이벤트를 실시간으로 전달해야 한다.

- 매칭 상태 변경
- 경기 시작
- 현재 게임 상태 스냅샷
- 합법 행동 성공
- 행동 실패 및 실패 사유
- 턴 변경
- 타이머 변경
- 카드 사용 결과
- 드로우 결과
- 체크, 체크메이트, 무승부, 시간패, 몰수패, 항복
- 채팅 메시지
- 재접속 상태

## 5. 매치메이킹

### 5.1 빠른 매치

- MVP는 빠른 매치만 제공한다.
- 친선 코드, 랭크 모드, 관전 모드는 제공하지 않는다.
- 사용자는 유효한 덱을 보유해야 빠른 매치 대기열에 진입할 수 있다.
- 매칭 전에는 사용자가 대기열을 취소할 수 있다.
- 매칭 확정 후 서버는 즉시 경기 생성 및 입장 상태로 전환한다.

### 5.2 매칭 기준

- 사용자는 비공개 단일 MMR을 가진다.
- 빠른 매치는 MMR이 가까운 상대를 우선한다.
- 대기 시간이 길어질수록 허용 MMR 범위를 점진적으로 넓힌다.
- 경기 결과에 따라 비공개 MMR을 갱신한다.
- 공개 티어나 랭크 표시는 MVP에 포함하지 않는다.
- 차단 관계가 있는 두 사용자는 빠른 매치에서 서로 매칭되지 않는다.

### 5.3 색 배정

- 서버가 경기 시작 시 백/흑을 무작위 배정한다.
- 백이 먼저 둔다.

## 6. 체스 룰

### 6.1 기본 규칙

- 보드는 표준 8x8 체스 보드다.
- 기물 종류, 수량, 초기 배치는 표준 체스를 따른다.
- 기본 승리 조건은 체크메이트다.
- 킹을 직접 잡는 방식으로 승리하지 않는다.
- 자기 턴에는 합법 수 하나를 반드시 둬야 한다.
- 둘 수 있는 합법 수가 없을 때는 체크메이트 또는 스테일메이트로 판정한다.

### 6.2 특수 수

MVP는 표준 체스 특수 규칙을 모두 지원한다.

- 캐슬링
- 앙파상
- 프로모션

프로모션은 폰이 끝 행에 도달한 즉시 플레이어가 퀸, 룩, 비숍, 나이트 중 하나를 선택해야 한다.

### 6.3 무승부

다음 표준 체스 무승부 조건을 지원한다.

- 스테일메이트
- 기물 부족
- 50수 규칙
- 3회 반복

플레이어 간 합의 무승부는 MVP에서 제외한다.

### 6.4 항복

- 플레이어는 경기 중 항복할 수 있다.
- 항복은 확인 단계를 거친 뒤 확정된다.
- 항복이 확정되면 즉시 해당 플레이어의 패배로 종료한다.

### 6.5 체스 룰 엔진

- 서버는 체스 룰 검증에 검증된 Java 라이브러리를 사용한다.
- 기본 후보는 `bhlangonijr/chesslib`이다.
- 카드 효과는 체스 라이브러리의 기본 합법 수 검증을 감싸는 별도 도메인 계층에서 처리한다.

## 7. 턴 진행

한 턴은 다음 순서로 진행한다.

1. 이동 전 카드 행동
2. 말 이동
3. 이동 후 행동
4. 턴 종료

### 7.1 이동 전 카드 행동

플레이어는 다음 중 하나를 선택한다.

- 카드 1장 사용
- 아무것도 하지 않음

이 단계에서 사용할 수 있는 카드는 카드 조건, 마나, 체크 상태 제한을 모두 만족해야 한다.

### 7.2 말 이동

- 말 이동은 항상 필수다.
- 서버는 이동의 합법성을 검증한다.
- 이동 결과 체크, 체크메이트, 무승부, 캡처, 프로모션, 앙파상 여부를 판정한다.
- 프로모션이 필요한 경우 승격 기물 선택이 완료되어야 다음 단계로 넘어간다.

### 7.3 이동 후 행동

플레이어는 다음 중 하나를 선택한다.

- 카드 1장 사용
- 일반 카드 1장 드로우
- 아무것도 하지 않음

이동 후 행동은 하나만 수행할 수 있다.

### 7.4 체크 상태의 카드 제한

- 자신의 킹이 체크 상태일 때는 체크 해소에 기여하는 카드만 사용할 수 있다.
- 카드 사용 후에도 최종적으로 킹이 체크 상태라면 해당 행동은 불법이다.
- 일반 카드와 레전드 카드 모두 이 제한을 따른다.

## 8. 시간 규칙

### 8.1 기본 시간

- 각 플레이어는 개인 시계 30분을 가진다.
- 자기 턴 동안에만 자신의 시간이 감소한다.
- 턴 시작 시 남은 시간이 10초 미만이면 10초로 보정한다.
- 남은 시간이 10초 이상이면 별도 보정하지 않는다.

### 8.2 시간패

- 서버 기준 개인 시간이 0초에 도달하면 3초 유예 시간을 둔다.
- 유예 시간 안에 유효한 행동이 완료되지 않으면 시간패 처리한다.
- 클라이언트 표시 시간보다 서버 시간이 우선한다.

### 8.3 연결 끊김

- 연결이 끊겨도 개인 시계는 계속 진행한다.
- 사용자가 20초 안에 재접속하면 현재 경기 상태로 복귀한다.
- 20초 안에 재접속하지 않으면 몰수패 처리한다.
- 연결 끊김으로 시간도 0초가 되면 시간패 또는 몰수패 중 서버가 먼저 확정한 종료 사유를 적용한다.

## 9. 카드 시스템

### 9.1 카드 종류

카드는 일반 카드와 레전드 카드로 나눈다.

- 일반 카드
  - 체스 룰을 깨지 않는 보조 효과 중심이다.
  - 상대에게 공개된다.
  - 경기 중 일반 카드 덱에서 드로우할 수 있다.
- 레전드 카드
  - 경기당 1장만 사용 가능한 1회성 궁극기다.
  - 사용 전까지 상대에게 공개되지 않는다.
  - 마나 외 추가 사용 조건을 가질 수 있다.
  - 전투 개입 효과를 가질 수 있으나 킹 직접 제거, 즉시 승리 효과는 MVP 카드 목록에서 제외한다.

### 9.2 덱 구성

- 한 경기용 덱은 레전드 카드 1장과 일반 카드 5장으로 구성한다.
- 레전드 카드는 덱에 지정된 1장이 게임 시작 시 확정 지급된다.
- 일반 카드는 경기 시작 시 덱에서 무작위 1장을 지급한다.
- 일반 카드 덱은 카드별 최대 2장까지 중복 편성할 수 있다.
- 덱 편성은 보유 카드 수량과 중복 제한을 모두 만족해야 한다.
- 유효하지 않은 덱으로는 빠른 매치에 참가할 수 없다.

### 9.3 경기 중 카드 공개 정보

- 자신의 모든 손패는 자신에게 보인다.
- 상대의 일반 카드 손패는 공개된다.
- 상대의 레전드 카드는 사용 전까지 비공개다.
- 일반 카드 덱 잔량은 상대에게 보인다.

### 9.4 드로우

- 일반 카드 드로우는 이동 후 행동의 선택지 중 하나다.
- 드로우 시 일반 카드 덱에서 1장을 뽑는다.
- 덱이 비어 있으면 드로우 행동은 실패한다.
- 드로우한 일반 카드는 상대에게도 공개된다.

### 9.5 카드 데이터 필드

카드는 최소한 다음 정보를 가진다.

- 카드 ID
- 카드명
- 카드 종류: 일반, 레전드
- 마나 비용
- 사용 가능 타이밍: 이동 전, 이동 후, 둘 다
- 대상 조건
- 추가 사용 조건
- 효과 타입
- 효과 파라미터
- 공개 여부
- 활성 여부

### 9.6 효과 범위

일반 카드의 효과 범위는 다음과 같이 제한한다.

- 특정 기물의 다음 이동 보조
- 특정 기물 보호
- 드로우 보조
- 마나 획득 또는 비용 감소
- 제한적인 정보 확인
- 체크 룰과 최종 킹 안전성을 깨지 않는 보조 효과

레전드 카드의 효과 범위는 다음을 포함할 수 있다.

- 상대 기물 제거
- 기물 위치 교환
- 특정 기물 강제 이동
- 특정 상태 해제
- 강한 보호 또는 반격 효과

레전드 카드라도 다음 효과는 MVP에서 만들지 않는다.

- 상대 킹 직접 제거
- 즉시 승리
- 표준 체스 승패 판정을 무시하는 효과

### 9.7 샘플 카드

샘플 카드는 구현 기준이 아니라 카드 구조 이해를 위한 예시다.

- 일반 카드 예시: 전술 정찰
  - 비용: 1마나
  - 타이밍: 이동 전 또는 이동 후
  - 효과: 상대의 공개 일반 카드와 일반 덱 잔량을 최신 상태로 다시 동기화하고, 자신의 다음 합법 수 후보를 서버 계산 기준으로 요청할 수 있다.
- 레전드 카드 예시: 왕실 친위대
  - 비용: 6마나
  - 추가 조건: 자신의 룩 1개 이상이 생존
  - 타이밍: 이동 전
  - 효과: 이번 턴 동안 지정한 아군 기물 1개가 카드 효과로 제거되지 않는다.

## 10. 마나 시스템

### 10.1 기본 규칙

- 각 플레이어의 시작 마나는 0이다.
- 최대 보유 마나는 10이다.
- 마나는 체스 행동 결과로 획득한다.
- 마나 획득은 서버가 이동 결과를 확정한 뒤 계산한다.

### 10.2 획득량

- 합법 이동: 1마나
- 캡처: 2마나
- 체크: 2마나
- 앙파상: 3마나

한 수가 여러 조건을 동시에 만족하면 모두 합산한다.

예: 이동하면서 캡처하고 체크까지 만들면 1 + 2 + 2 = 5마나를 얻는다.

최대 보유량을 초과하는 마나는 버린다.

## 11. 카드팩 및 컬렉션

### 11.1 골드

- 서비스 재화 이름은 골드다.
- MVP에서는 골드 잔액 필드만 준비한다.
- 카드팩 가격은 0골드로 설정한다.
- 출석 보상, 경기 보상, 골드 구매는 MVP 이후 확장 대상으로 둔다.

### 11.2 카드팩

- 카드팩 1개는 카드 3장을 지급한다.
- 각 카드 슬롯은 독립적으로 레전드 여부를 판정한다.
- MVP의 레전드 확률은 각 슬롯당 1%다.
- 따라서 카드팩 1개에서 레전드 카드가 0장, 1장, 2장, 3장 모두 나올 수 있다.
- 레전드가 아닌 슬롯은 일반 카드로 지급한다.

### 11.3 중복 카드

- 이미 보유한 카드를 다시 얻으면 보유 수량이 증가한다.
- 중복 카드를 자동 환급하거나 결과를 보정하지 않는다.
- 덱 편성 시 보유 수량과 덱 중복 제한을 함께 검증한다.

### 11.4 스타터 컬렉션

- 신규 사용자는 즉시 플레이 가능한 스타터 덱을 지급받는다.
- 스타터 덱은 기본 레전드 카드 1장과 일반 카드 5장으로 구성한다.

### 11.5 카드팩 감사 로그

카드팩 개봉 결과는 DB에 기록한다.

- 사용자
- 개봉 시각
- 사용 골드
- 결과 카드 3장
- 각 카드의 일반/레전드 여부
- 확률 버전
- 중복 여부

## 12. 전적 및 기보

### 12.1 전적 요약

마이페이지에서는 다음 정보를 제공한다.

- 총 경기 수
- 승리 수
- 패배 수
- 무승부 수
- 승률
- 최근 경기 목록

최근 경기 목록은 최소한 다음 정보를 포함한다.

- 상대 닉네임
- 백/흑 여부
- 경기 시작 시각
- 경기 종료 시각
- 종료 사유
- 결과

### 12.2 기보 저장

- 경기의 전체 수순을 저장한다.
- 카드 사용 로그를 저장한다.
- 마나 변화 로그를 저장한다.
- 주요 상태 변화 로그를 저장한다.
- 기보는 MVP에서 해당 경기 당사자만 조회할 수 있다.
- 공개 기보 기능은 이후 확장 대상으로 둔다.

## 13. 대전 채팅

### 13.1 기본 정책

- 대전 중 두 플레이어는 자유 텍스트 채팅을 사용할 수 있다.
- 채팅 메시지는 경기 채널에 실시간 전달된다.
- 채팅 로그는 7일간 보관한다.
- 차단한 사용자의 채팅은 표시하지 않는다.

### 13.2 금칙어 처리

- 서버는 기본 금칙어 필터를 가진다.
- 사용자에게 송출되는 채팅 메시지는 금칙어가 마스킹된 내용이다.
- DB에는 신고 검토를 위해 원문 메시지를 저장한다.
- 관리자 화면에서는 원문과 마스킹 결과를 검토할 수 있어야 한다.

## 14. 커뮤니티

### 14.1 접근 권한

- 커뮤니티 조회와 작성은 모두 로그인 사용자만 가능하다.
- 계정정지 사용자는 커뮤니티를 사용할 수 없다.
- 채팅정지 사용자의 커뮤니티 작성 제한 여부는 제재 정책에 따른다.

### 14.2 게시글

게시글은 다음 기능을 제공한다.

- 작성
- 목록 조회
- 상세 조회
- 수정
- 삭제
- 검색
- 신고
- 좋아요 토글

이미지 업로드는 MVP에서 제외한다.

### 14.3 댓글

댓글은 다음 기능을 제공한다.

- 작성
- 조회
- 삭제
- 신고

댓글 좋아요는 MVP에서 제외한다.

### 14.4 검색

- 제목, 내용, 작성자 닉네임을 대상으로 검색한다.
- MVP는 DB 단순 검색을 사용한다.
- 전문 검색 엔진 또는 고급 랭킹은 이후 확장 대상으로 둔다.

### 14.5 좋아요

- 좋아요는 게시글에만 적용한다.
- 로그인 사용자는 게시글마다 좋아요를 한 번 누를 수 있다.
- 이미 누른 좋아요는 취소할 수 있다.

### 14.6 신고 및 자동 숨김

- 게시글과 댓글을 신고할 수 있다.
- 신고가 5건 누적되면 자동 임시 숨김 처리한다.
- 관리자는 신고 내용을 검토하고 숨김 유지, 복구, 사용자 제재를 결정한다.

### 14.7 커뮤니티 금칙어 처리

- 게시글과 댓글 원문은 DB에 저장한다.
- 사용자 화면에는 금칙어가 마스킹된 내용을 표시한다.
- 관리자 검토 시 원문을 확인할 수 있다.

## 15. 신고, 차단, 관리자

### 15.1 사용자 차단

- 사용자는 다른 사용자를 차단할 수 있다.
- 차단 관계가 있는 사용자는 빠른 매치에서 서로 매칭되지 않는다.
- 차단한 사용자의 대전 채팅은 표시하지 않는다.

### 15.2 신고

신고 대상은 다음을 포함한다.

- 대전 채팅
- 커뮤니티 게시글
- 커뮤니티 댓글
- 사용자 프로필

신고는 관리자 검토 대상으로 저장한다.

### 15.3 관리자 기능

MVP는 최소 관리자 기능을 제공한다.

- 신고 목록 조회
- 신고 상세 조회
- 게시글 숨김 및 복구
- 댓글 숨김 및 복구
- 사용자 채팅정지
- 사용자 계정정지
- 카드 목록 조회

카드 효과, 확률, 팩 가격 수정은 MVP 관리자 기능에 포함하지 않는다.

### 15.4 제재

- 채팅정지는 기간제로 적용한다.
- 계정정지도 기간제로 적용한다.
- 계정정지 사용자는 로그인 이후 주요 기능 사용이 제한된다.
- 관리자 권한은 초기에는 DB에서 수동 부여한다.

## 18. 데이터 저장 원칙

### 18.1 경기 상태

- 활성 경기 상태는 메모리에 유지한다.
- 주요 이벤트와 스냅샷은 DB에 저장한다.
- 서버 재시작 시 완전한 활성 경기 복구는 MVP 필수 요구사항이 아니다.
- 단, 종료된 경기의 결과와 기보는 DB에 보존되어야 한다.

### 18.2 개인정보 및 로그

- 채팅 로그는 7일 보관 후 삭제 또는 비식별화한다.
- 신고 검토에 필요한 원문은 보관 기간 안에서 관리자만 접근할 수 있다.
- 탈퇴 사용자의 기록은 탈퇴 사용자로 표시한다.

## 19. 핵심 수용 기준

### 19.1 인증

- 사용자는 이메일/비밀번호로 가입할 수 있다.
- 이메일 인증 전 사용자는 주요 기능을 사용할 수 없다.
- Google 또는 Kakao OAuth로 로그인할 수 있다.
- 인증된 세션으로 REST API와 STOMP WebSocket에 접근할 수 있다.
- 로그아웃 후 기존 세션으로 보호 API를 사용할 수 없다.

### 19.2 매칭

- 유효 덱을 가진 사용자는 빠른 매치 대기열에 진입할 수 있다.
- 유효 덱이 없으면 대기열 진입이 실패한다.
- 두 사용자가 매칭되면 서버가 백/흑을 랜덤 배정하고 경기를 생성한다.
- 차단 관계가 있는 사용자는 서로 매칭되지 않는다.

### 19.3 게임 진행

- 서버는 표준 체스 합법 수만 허용한다.
- 자기 턴에는 이동 전 카드 행동, 필수 말 이동, 이동 후 행동 순서가 지켜진다.
- 이동 후 행동에서는 카드 사용, 드로우, 아무것도 안 함 중 하나만 수행된다.
- 체크 상태에서는 체크 해소에 기여하지 않는 카드 행동이 거부된다.
- 체크메이트, 스테일메이트, 50수 규칙, 3회 반복, 기물 부족이 올바르게 판정된다.

### 19.4 마나 및 카드

- 이동, 캡처, 체크, 앙파상 마나가 합산 지급된다.
- 마나는 10을 초과해 저장되지 않는다.
- 레전드 카드는 경기 시작 시 지정된 1장이 지급된다.
- 일반 카드는 시작 시 무작위 1장이 지급된다.
- 일반 카드 손패는 상대에게 공개된다.
- 레전드 카드는 사용 전까지 상대에게 비공개다.

### 19.5 시간 및 재접속

- 각 플레이어는 개인 시계 30분을 가진다.
- 자기 턴 동안에만 시간이 감소한다.
- 턴 시작 시 남은 시간이 10초 미만이면 10초로 보정된다.
- 시간이 0초가 된 뒤 3초 안에 유효 행동이 없으면 시간패 처리된다.
- 연결 끊김 후 20초 안에 복귀하면 경기 상태가 재동기화된다.
- 20초 안에 복귀하지 않으면 몰수패 처리된다.

### 19.6 카드팩

- MVP 카드팩은 0골드로 구매할 수 있다.
- 카드팩 1개는 카드 3장을 지급한다.
- 각 카드 슬롯은 독립적으로 레전드 1% 확률을 적용한다.
- 중복 카드는 보유 수량으로 누적된다.
- 카드팩 개봉 로그가 저장된다.

### 19.7 커뮤니티 및 운영

- 로그인 사용자는 게시글과 댓글을 작성할 수 있다.
- 게시글 검색이 동작한다.
- 게시글 좋아요는 토글 가능하다.
- 신고 5건이 누적된 게시글 또는 댓글은 자동 임시 숨김 처리된다.
- 관리자는 신고를 검토하고 숨김, 복구, 제재를 적용할 수 있다.
- 금칙어가 포함된 채팅, 게시글, 댓글은 사용자 화면에 마스킹되어 표시된다.
- 관리자 검토용 DB 데이터에는 원문이 보존된다.
